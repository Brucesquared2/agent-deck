# Memory Map

> **Start here.** This file explains the memory layer structure and how to navigate it.

## Layer Overview

| Layer | Folder | Purpose | Writers |
|-------|--------|---------|---------|
| 00 · Core | `00-core/` | Principles, north star, policies | Planner only |
| 10 · Ops | `10-ops/` | Runbooks, checklists, incident notes | Any agent |
| 20 · Projects | `20-projects/` | Per-project packs | Planner + Implementer |
| 30 · Sessions | `30-sessions/` | Per-session logs and summaries | Any agent |
| 40 · Knowledge | `40-knowledge/` | Facts, references, learned patterns | Any agent |
| 50 · Maps | `50-maps/` | Indexes and maps (this folder) | `refresh-indexes.sh` only |
| 60 · Signals | `60-signals/` | Signal-lang spec + examples | Any agent (blocks only) |
| 90 · Archive | `90-archive/` | Retired entries | Planner only |

---

## Navigation Quick-Start

1. **New session?** → run `scripts/new-session.sh <project-slug>` then open the generated file in `30-sessions/`.
2. **Looking for a decision?** → `50-maps/DECISION_LOG.md`
3. **Looking for a tag or topic?** → `50-maps/TAG_INDEX.md`
4. **Looking for a person/system?** → `50-maps/ENTITY_INDEX.md`
5. **Looking for all files?** → `50-maps/INDEX.md`
6. **Need to emit a signal?** → `60-signals/signal-lang.md` for the spec, `60-signals/examples.md` for examples.

## Maintenance

- Run `scripts/refresh-indexes.sh` after any batch of writes.
- Use `scripts/new-session.sh <project-slug>` to create a session file from the template.
- Move stale entries to `90-archive/` via Planner agent after 90 days of inactivity.
